On 2/10/14 11:57 AM, Waltermire, David A. wrote:

Blibbet,

I don't think we can simply ignore SWIDs. In my personal opinion, SWIDs have many advantages over CPEs including:

- Providing authoritative, high-fidelity software data generated at build time by the software creator/publisher. CPEs are commonly generated by 3rd parties in a labor intensive way by scraping vendor websites and vulnerability bulletins for software data that is often incomplete and inaccurate.
- Providing more robust software metadata that goes beyond the 11 fields supported by CPE including tags/identification for patches which CPE doesn't support.
- Normalizing the data representation for installed software on endpoints regardless of platform/software specific technologies.
- Providing software footprint details including composite files, versions, and hashes. This is not something that CPE supports.

Simply put, SWIDs provide greater support for the management of software across the deployment lifecycle than CPE allows for. This has both security and IT operational benefits. SWIDs allow for greater coverage and address the scalability concerns that currently exist with CPE.

With SWID there is not a requirement to pay or join any trade group to participate. Software publishers are responsible for the creation and propagation of SWID tags they produce. There is not a requirement to centrally curate them like there is with the US Government curated CPE dictionary.

The ISO specification is behind a paywall, which is the ISO model for publication. The IETF and ISO both reference each other's specifications. This should not be a significant barrier.

The schema is freely available. You can access it here:

http://standards.iso.org/iso/19770/-2/2009/schema.xsd

TagVault is developing best practice guides and certification requirements to help interested software publishers to demonstrate the degree of information provided by a publisher's SWID tags to support various use cases. They have stated publically that they intend to make these documents publically available as they are produced. Participation in TagVault is not a requirement for SWID tag publication; it is simply a value added service for publishers and consumers if they choose to participate.

I know of no US$500/year cost for use of CPE. Current CPE information is provided for use on a voluntary basis at no cost to the consumer at:

http://nvd.nist.gov/cpe.cfm

I hope this helps address your questions/concerns.

Sincerely,
Dave
